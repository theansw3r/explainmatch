# Contributing — ExplainMatch

## Flujo de trabajo
- Las tareas se gestionan en GitHub Issues y Project Board.
- Toda entrega de código se hace vía Pull Request (PR) cuando el equipo trabaje en local.

## Convenciones
- Issues:
  - Funcionales: `HUxx - ...`
  - No funcionales: `HU NFxx - ...`
  - Incluir: SP, responsable, criterios de aceptación (GWT) y checklist mínimo en Sprint 1.
- Commits (sugerencia):
  - `feat: ...` nueva funcionalidad
  - `fix: ...` corrección
  - `docs: ...` documentación
  - `chore: ...` tareas de mantenimiento

## Definition of Done (DoD)
Una historia está "Done" cuando:
- Cumple criterios de aceptación
- Tiene evidencias (captura/endpoint/UI) según aplique
- Está documentada (README / docs)
